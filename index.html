MapReduce-osdi04 Paper Read

发表于 2026-06-03

MapReduce: 大规模集群上的简化数据处理

  • 原始论文

摘要

用户侧

  • map 函数：处理键值对，生成一组中间键值对
  • reduce 函数：合并与同一中间键关联的所有中间值

特性

  1. 自动并行化，适于在大型机器集群上执行。
  2. 运行时系统负责处理输入数据的分区、跨机器集合调度程序执行、处理机器故障以及管理所需机器间通信。提供了一种方法，封装了分布式系统的复杂性。
  3. 具有高扩展性，可以适配多种分布式程序任务

引言

MapReduce框架的出现，是为了解决，大量数据如何在大型分布式系统上实现并行化计算、分布数据和故障处理等，复杂的并发细节（该问题导致原来设计的简单计算任务代码，为了满足分布式系统的并发需求，而增加了大量的并发复杂性）

MapReduce在计算任务与分布式系统之间设计了一种新的抽象，将简单计算与复杂并发细节相分离。

MapReduce库的抽象设计受到Lisp及许多函数式语言中的 map 与 reduce 原语的启发。对输入的每个输入逻辑应用 map 操作，计算一组中间键值对，对共享相同键的值应用 reduce 操作，以适当的合并派生数据。将函数式的 map 与 reduce 的实现交由用户制定，库只需要提供并行化方法，使用户轻松实现程序并行化。该库将重新执行作为容错的主要机制。

map 与 reduce

  • map 映射
  • reduce 归约

SFML-cloth-simulation

发表于 2026-06-02

SFML cloth simulation

使用 粒子弹簧模型 进行 运动学模拟

理论公式

位置 Verlet 积分

选用 Verlet 位置更新公式

Verlet 积分: x(t + dt) = 2x(t) - x(t - dt) + a(t)dt^2 [O(dt^4)]

公式推导

泰勒展开法

更新质点状态

  1. 假设粒子运动状态

    • 粒子的位置为 x(t)
    • 速度为 v = x’(t)
    • 加速度为 a(t) = F(x(t)) / m = x’’(t)
    • 以上三项公式都仅依赖于 位置

  2. 取时间步长为 dt ，分别在 t 时刻前后作三阶泰勒展开

    • x(t + dt) = x(t) + x’(t)dt + (1/2)x’’(t)dt^2 + (1/6)x’’’(t)dt^3 [+ O(dt^4)]
    • x(t - dt) = x(t) - x’(t)dt + (1/2)x’’(t)dt^2 - (1/6)x’’’(t)dt^3 [+ O(dt^4)]

  3. 将上述两式相加

    • x(t + dt) + x(t - dt) = 2x(t) + x’’(t)dt^2 [+ O(dt^4)]
    • 将上式进行整理 x(t + dt) = 2x(t) - x(t - dt) + x’’(t)dt^2 [+ O(dt^4)] 得到上述公式
    • 可见，该式表示，当前的位置仅依赖于上一时刻的位置，以及当前的加速度
    • 若需要速度，可是使用 中心差分估计
      • v(t) = ( x(t + dt) - x(t - dt) ) / 2dt [+ O(dt^2)]

Raylib 杂记

发表于 2026-05-29

Raylib 杂记

Raylib 简介
Raylib 是一个 简单、易用且跨平台 的游戏开发库，无外部依赖，
特别适合用来学习图形学、游戏编程和快速制作原型。
它完全使用 C 语言写成（同时也提供了超过 50 种语言的绑定），
核心设计理念是“没有复杂依赖，开箱即用”。

2D坐标，原点在屏幕左上角

  • x: 指向屏幕横向向右
  • y: 指向屏幕纵向向下

3D坐标，采用右手系

  • x: 指向屏幕横向向右
  • y: 指向屏幕纵向向上
  • z: 指向屏幕外侧

OpenGL 相关模块

GLFW

Graphics Library Framework
是一个专门为 OpenGL 设计的 跨平台窗口和输入处理库

  • 功能：创建窗口、创建 OpenGL 上下文、处理键盘鼠标手柄输入、管理时间等，彻底屏蔽操作系统层面的差异
  • 目前版本的 Raylib 不再依赖 GLFW，目前Raylib已拥有窗口和输入管理功能

GLSL

OpenGL Shading Language
是运行在 GPU 上的， 着色器编程语言，语法类似于 C

  • 作用：现代图形渲染管线是“可编程”的，GLSL 就是用来编写这些运行在显卡上的程序。最常见的是顶点着色器（处理每个顶点的位置变换）和片元着色器（计算每个像素的颜色）。
  • 使用前提：将 顶点着色器 放在 .vs 命名的目录下，将 面着色器 放在 .fs 命名的目录下

在 Raylib 中可以使用 LoadShader() 加载 <name>.glsl 文件，并使用 BeginShaderMode() 应用该着色器程序

<rlgl.h>

Raylib OpenGL

对 OpenGL 渲染管线 进行了轻量封装，提供了一个操作 OpenGL层 的方法，是 Raylib 与 OpenGL 的一个中间层

API与数据类型设计

Function 函数

每个 Raylib 函数命名都遵循以下语法结构：
<动词><主语><补语>
或者说
<动作行为><对象><属性/状态>
即：
在 <某物> 的 <某 属性/状态> 上，做 <动作行为>

Raylib 函数大多被设计为值传递函数，小部分需要传递引用

Lists 列表

  • cons 本质是一个二元组
    • 形式为 '(1 2)
    • 输出形式为 (1 . 2) 使用 . 进行分隔
  • list 表示一串二元组嵌套而成的列表
    • 形式为 '(1 2 3 4 5)
    • 本质是 '(cons 1 (cons 2 (cons 3 (cons 4 '())))) 最内层的后一个元素（list最后一个元素之后）为 空 '()
    • 输出形式为 (1 2 3 4 5) 将空元素省略
    • 真列表(proper list)的结尾为一个隐式的 '() 元素

这也就可以解释，为什么一下用例中 (list? cons-cons) 产生 #f 返回，因为， cons-cons 就不是真列表，而是一个 (cons (list ...) elem) 的二元组

而 (cons? ...) 只需要检测输入是否为二元组，并且检查时直接将 list 展开为 cons 在下列用例中无论如何都返回为 #t

相当于 (cons 1 (list 2 3 4 5)) 相当于 (list 1 2 3 4 5) 将cons内部的list展开
