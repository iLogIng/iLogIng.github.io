关于JavaScript特性与异步模型

执行上下文

JS在第一次执行时会根据web引擎创建 全局上下文 在该上下文中运行JS脚本，在 调用函数时 从内存堆中提取函数 创建该函数的函数执行上下文 再将程序的执行转入 该函数上下文中 函数执行后再 返回全局上下文（这与C/C++的函数栈帧不同，反倒与thread机制类似，即进行上下文的保存和转换， 据此可推知： JS所实现的单线程异步是以此为基础实现的）

  • 关于JS中函数的存储

    是的，JS将定义的函数及其形参都按一定规则存入内存堆，在调用时从堆中提取。

调用栈

调用栈是JS引擎 跟踪 被调用函数所处位置的方式，JS使用其管理 执行上下文。

JS为每个被调用的函数创建一个 函数上下文 并依次压入调用栈中，由于JS为单线程语言，所以 只维护一个调用栈 当栈空后 脚本停止 。当执行的是一个递归函数，且陷入无限递归时会产生 栈溢出。

由于JS为单线程语言，只维护一个调用栈，所以 一次只执行一个操作 所有操作都是 同步的 而JS的 异步 特性可以在等待另一个任务完成时，执行其他任务。

事件循环

JS引擎从文件顶部向下执行，创建 执行上下文 和 管理 调用堆栈

在程序运行时，有可能会遇到执行时间很长的 阻塞函数 会阻塞交互行为，

所以使用Web API传入回调函数，函数向宿主环境发送任务消息后返回上一层上下文继续执行，其任务交由宿主环境进行处理，再接收宿主环境返回的完成信息（包含之前所注册的回调函数），压入 任务队列 （或回调队列），再由 事件循环 检查任务队列中的内容，当 调用栈 和 任务队列 都为空后 脚本停止 。

RDI 目的索引 (Destination Index) 第1个整数参数、字符串/数组操作目的地址

计算机网络概观

发表于 2026-03-28

基于 《计算机网络：自顶向下方法》

因特网是一个世界范围内的计算机网络

入网设备 被称为 主机(host) 或 段系统(end system)

端系统通过

  • 通信链路 (communication link)
  • 分组交换机 (packet switch)

的网络连接在一起。

通信链路 是一组物理媒介，是信息流的载体。

链路 有由其性质决定的 传输速率(transmission rate) 以 bit/s (bps) 为度量。

网络中的信息是以信息包的形式被 分组(packet) 流通的，从始端由网络传送至目的端，并在目的端装配复原。

当今，因特网中有两种常见的交换机：

  • 路由器 (router) 通常用于网络核心中
  • 链路层交换机 (link-layer switch) 通常用于接入网中

二者的功能都是向目的端转发分组。

路径 (route / path) 从 发送端系统 到 接收端系统 ，一个分组所经历的 一系列通信链路 和 分组交换机

端系统 通过 因特网服务商 (Internet Service Provider , IPS) 接入因特网。

端系统，分组交换机和其他英特网组件都要运行多个 协议（protocol） 这些协议控制英特网中信息的接收和发送。

因特网的主题协议统称为 TCP/IP

  • TCP (Transmission Control Protocol, 传输控制协议)
  • IP (Internet Protocol, 网际协议) 定义了在路由器和端系统之间，发送和接收分组的格式。

因特网标准 (Internet Standard) 由 因特网工程任务组 (Internet Engineering Task Fore, IETF) 研发。其标准文档称为 请求评论 (Requst For Comment, RFC)

涉及多个相互交换数据的应用称为 分布式应用 (distributed application) 。（如：社交媒体、多人游戏、交通信息… …）

与因特网相连的端系统提供了一个 套接字接口 (Socket interface) 套接字接口规定了运行在一端系统上的程序 请求 因特网基础设施向运行在另一台端系统上的 特定目的程序 交付数据 的方式，是入网设备必须遵守的规则集合。

协议（protocol）

因特网广泛使用协议，不同协议用于完成不同的通信任务
为完成一项工作，要求两个（或多个） 通信实体 运行相同的协议。

  • 协议 定义了在两个（或多个）通信实体之间交换报文的格式和顺序 ，以及报文的 发送/接收 或遇到其他事件时应采取的操作。

边缘网络

端系统身处因特网边缘，故有此称谓。

端系统，因其容纳应用程序，故可称作 主机（host），分为两类：

  • 客户端（client）
  • 服务器（server）

接入网： 将端系统物理连接到 边缘路由器 (Edge router) 的网络。该路由器是端系统接入网络后的第一台路由器。

对于每个 传输器-接收器 对，一比特(1 bit)通过传播电磁波或光脉冲的方式，跨越物理媒介 (physical medium) 进行发送。且对于路径上的每个 传输器-接收器 对而言，不必具有相同类型。
物理媒介分为两种类型：

  • 导引型媒介 (guided media)
  • 非导引型媒介 (unguided media)

物理链路的成本与其他网络成本相比是最小的，但是其安装与铺设成本才是其昂贵的原因。

网络核心

端系统间彼此交换 报文 (message) ，该报文可以 执行某种控制功能 或 承载某些数据。

为了从 源端系统 向 目标端系统 发送一个 报文 ，源将 长报文 划分为较小的数据块，这一划分报文，使其适于网络传送的过程成为 分组 (packet) 。在源于目的之间，每个分组都通过 通信链路 和 分组交换机 (packet switch) 传送，交换主机主要有 路由器 (router) 和 链路层交换机 (link-layer switch) 两类。

文分组以 等于 该链路最大传输速率的速度通过通信链路。即，若发送一个 *L bit 的分组，链路的传输速率为 R bit ，则分组通过链路的时间为 L/R sec

多数交换机在 输入端 使用 存储转发器 (store-and-forward transmission) 机制。先缓存一个接收的分组后，再将其发出。
即，接收-转发 时间延迟为：

  • d_{end-end} = N * L/R

    总时间延迟为：
    L/R + (N-2) * L/R + L/R = N * L/R
    P 个分组经过N条链路的时间延迟为 (N + P - 1) * L/R

每台交换机有多条链路与之相连，对于每条相连的链路，该分组交换机具有一个 （输出缓存, output buffer 也称为 输出队列 output queue） 用于存储路由器准备发往某条链路的分组。

即，一个分组在链路中存在：

  • 转发时延
  • 排队时延 (queuing delay)
    在缓存占满后，将出现 分组丢失 (丢包 packet loss) 问题，从到达分组或已排队分组之一选择丢弃。

因特网中所采用的转发方式

根据网络包中的 源端系统 的 IP地址 ，相同一台 相邻路由器 进行转发，由每台路由器自己维护 转发表 (forwarding table) 进行控制。每个分组包括每台路由器在传输中都不知道某一个分组在网络中的完整路径，路由器仅知道某个分组发往的下一台 路由器/交换机（网络节点）。

路由器的 转发表 (forwarding table) 由 路由选择协议 (routing protocol) 来规范其 自动设置转发表的行为。

以上为 分组交换 (packet switching) 机制

  • 频分复用 (Frequency-Division Multiplexing, FDM)
    • 链路的频谱由跨越链路创建的所有连接共享。特别的，在连接期间，链路对每条连接专门设定一个 频段 （电话网络中通常是 4Hz 宽）。该频段宽度就是 带宽 (bandwidth) 链路频谱中按带宽分有多个不同的频段。（每条链路持续得到部分带宽）
  • 时分复用 (Time-Division Multiplexing, TDM)
    • 时间被划分为固定时段的 帧 ，每个帧有被划分为固定数量的 时隙 。网络跨越一条链路，创建一条链接，网络在每个帧中为该连接指定一个时隙。
    • [帧 是一系列任务的集合] 每条电路得到一个时隙，在 **静默期（silent period）**专用电路处于空闲状态，并不被利用。

段系统经由一个 接入 ISP 与因特网相连，但为使各入网端系统互联互通，则需要 接入ISP 自身互联 组建 网络的网络（因特网）

因特网网络结构的构成发展

  1. 网络结构 1
    • 使用单一的 全球传输ISP(供应商) 互联所有的 接入ISP(客户)
  2. 网络结构 2 两层结构
    • 多个公司建立并运营各自的全球传输LSP，相互竞争各自争取接入ISP。且多个全球传输ISP之间，必须互联。
  3. 网络结构 3 等级化网络结构（两层客户端-供应端关系）
    • 任何给定的区域有其自身的 区域ISP(reginal ISP) 。每个区域ISP都与 第一层ISP(tier-1 ISP) 相连。（第一层ISP位于该等级结构的顶部）
  4. 网络结构 4
    • 在等级话网络结构3上增加， 存在点(Point of Presence, PoP) 、 多宿(multi-home) 、 对等(peer to peer) 、 英特网交换点(Internet Exchange Point, IXP) 构成的一个生态系统
    • PoP
      • 存在于等级结构的 所有层次 中,底层（接入ISP）除外。PoP是 供应商网络 中的一台或一组路由器（在相同位置）群组。其中， 客户ISP 与 供应商ISP 连接，从第三方电信供应商租用高速链路并将其路由器之一直接与PoP的一台路由器相连
    • 多宿
      • 除第一层ISP，其他任何ISP都可以选择多宿，与 多个供应商ISP 相连，以保障其与其它网络的连接
    • 对等
      • 不与上层供应商产生通信流量，而是与 平级ISP 相连，将两个平级ISP的网络相连，跨网传输不需要向上层供应商ISP产生路由成本。平级ISP间传输，无流量结算。
    • IXP
      • 是一个汇合点，多个ISP可以在此处一起对等
  5. 网络结构 5
    • 在网络结构4的顶部增加 内容供应商网络(content provider network) 构建而成

网络时延

在分组从一台主机经历通信链路传输到另一台主机的过程中，其必定在每个节点都会受到集中不同类型的时延制约。

  • total modal delay
    • 节点处理时延(nodal processing delay)
    • 排队时延(queuing delay)
      • 是最复杂的成分，受某时刻的网络环境影响
      • 分组大小，到达顺序，相对位置等都会产生影响
      • 在表征排队时延时，通常使用统计量进行度量
    • 传输时延(transmission delay)
      • 节点向链路发送整个分组的时间
    • 传播时延(propagation delay)
      • 分组在两个节点间链路中传播的时间，受制于传播媒介的物理性质
    • …

协议栈

OSI网络分层(7层)

  • 应用层
  • 表示层
  • 会话层
  • 传输层
  • 网络层
  • 数据链路层
  • 物理层

目前常使用的是 5层 协议栈（TCP/IP协议簇）

  1. 应用层
    • 网络应用程序和应用层协议所在之处，位于应用层的消息分组成为 报文(message)
  2. 运输层
    • 英特网中的运输层在程序端点之间传送应用层报文，位于运输层的分组称为 报文段(segment)
  3. 网络层
    • 英特网的网络层负责将称为 数据报(datagram) 的网络层分组从一台主机传输到另一端主机
  4. 链路层
    • 为了将分组从一个节点移动到路径上的下一个节点，网络层必须依靠 链路层的服务
    • 链路层提供的服务取决于应用于该链路的特定的链路层协议
    • 链路层的分组称为 帧(frame)
  5. 物理层
    • 将 帧 中的一个个 bit 从一个节点移动到下一个节点

在各个层中，使用上一层传来的分组，加上该层的 首部 ，即， 首部字段 + 有效载荷字段(来自上一层的分组)
