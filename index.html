MapReduce 程序设计

原始论文

实验目标

  • 学习 MapReduce 范式的通用特性
  • 使用线程和相关函数实现一个正确且高效的 MapReduce 框架
  • 获得更多编写并发代码的经验

MapReduce框架总体思路

用户只需要定义 Map() 与 Reduce() 函数，交由框架处理该函数指针即可

  • Map(): 接受一个输入对，并产生一组中间键值对。
  • Reduce(): MapReduce库 将具有相同中间键K的中间值分组在一起，传入该函数
    • 接受一个中间键K和该键的一组值。将这些值进行合并为更小的集合
    • 通常，每次对 Reduce() 的调用产生零个或一个输出值
    • 中间值通过迭代器提供给用户的reduce函数

MapReduce 可将许多不同类型的相关计算映射到该框架上。
并且多个 mapper 可以同时运行，多个 reducer 也可以同时运行。
用户不必亲自操作编写并行化程序，只需要编写 Map() 与 Reduce() 函数，并交由该框架提供的相关基础设施即可。

MapReduce-osdi04 Paper Read

发表于 2026-06-03 更新于 2026-06-04

MapReduce: 大规模集群上的简化数据处理

  • 原始论文

摘要

用户侧

  • map 函数：处理键值对，生成一组中间键值对
  • reduce 函数：合并与同一中间键关联的所有中间值

特性

  1. 自动并行化，适于在大型机器集群上执行。
  2. 运行时系统负责处理输入数据的分区、跨机器集合调度程序执行、处理机器故障以及管理所需机器间通信。提供了一种方法，封装了分布式系统的复杂性。
  3. 具有高扩展性，可以适配多种分布式程序任务

方案背景

MapReduce框架的出现，是为了解决，大量数据如何在大型分布式系统上实现并行化计算、分布数据和故障处理等，复杂的并发细节（该问题导致原来设计的简单计算任务代码，为了满足分布式系统的并发需求，而增加了大量的并发复杂性）

MapReduce在计算任务与分布式系统之间设计了一种新的抽象，将简单计算与复杂并发细节相分离。

MapReduce库的抽象设计受到Lisp及许多函数式语言中的 map 与 reduce 原语的启发。对输入的每个输入逻辑应用 map 操作，计算一组中间键值对，对共享相同键的值应用 reduce 操作，以适当的合并派生数据。将函数式的 map 与 reduce 的实现交由用户制定，库只需要提供并行化方法，使用户轻松实现程序并行化。该库将重新执行作为容错的主要机制。

  • map 映射
  • reduce 归约

SFML-cloth-simulation

发表于 2026-06-02

SFML cloth simulation

使用 粒子弹簧模型 进行 运动学模拟

理论公式

位置 Verlet 积分

选用 Verlet 位置更新公式

Verlet 积分: x(t + dt) = 2x(t) - x(t - dt) + a(t)dt^2 [O(dt^4)]

公式推导

泰勒展开法

更新质点状态

  1. 假设粒子运动状态

    • 粒子的位置为 x(t)
    • 速度为 v = x’(t)
    • 加速度为 a(t) = F(x(t)) / m = x’’(t)
    • 以上三项公式都仅依赖于 位置

  2. 取时间步长为 dt ，分别在 t 时刻前后作三阶泰勒展开

    • x(t + dt) = x(t) + x’(t)dt + (1/2)x’’(t)dt^2 + (1/6)x’’’(t)dt^3 [+ O(dt^4)]
    • x(t - dt) = x(t) - x’(t)dt + (1/2)x’’(t)dt^2 - (1/6)x’’’(t)dt^3 [+ O(dt^4)]

  3. 将上述两式相加

    • x(t + dt) + x(t - dt) = 2x(t) + x’’(t)dt^2 [+ O(dt^4)]
    • 将上式进行整理 x(t + dt) = 2x(t) - x(t - dt) + x’’(t)dt^2 [+ O(dt^4)] 得到上述公式
    • 可见，该式表示，当前的位置仅依赖于上一时刻的位置，以及当前的加速度
    • 若需要速度，可是使用 中心差分估计
      • v(t) = ( x(t + dt) - x(t - dt) ) / 2dt [+ O(dt^2)]
