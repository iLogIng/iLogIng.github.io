kv程序设计

程序要求

命令设计

  • put：

    格式: p,key,value

    • 其中 key 是 整数 ，value 是 任意字符串

  • get：

    格式: g,key

    • 其中 key 是整数。
      • 键存在：系统应打印出key,value \n
      • 键不存在：单独一行打印错误消息，格式为 K not found，其中 K 是键的实际整数值

  • delete：

    格式：d,key

    • 功能：
      • 删除相应的键值对（不打印任何内容）
      • 删除失败（并打印 K not found，其中 K 是键的实际值，即某个整数）。

  • clear：

    格式: c

    • 功能：从数据库中移除所有键值对。

  • all：

    格式: a

    • 功能：任意顺序打印数据库中的所有键值对，每行一个键值对 key,value

  • 文件操作

    • fopen
    • fwrite
    • fread
    • getline
    • fprintf

  • 字符串处理

    • strsep(char** line, char** delim)

      char** token = strsep(char** line, char** delim)
      按 delim 分割 *line

      • 返回 顺序第一个delim前的 token
      • *line 指向 顺序第一个delim后的 token
      • 相当于 原 *line = token append 后*line

reverse

reverse 程序的功能分别是进行文件的压缩(wzip)与解压缩(wunzip)
使用 游程编码(REL) 进行压缩

依赖例程

  • fopen()

  • fgets()

  • fclose()

  • getline()

  • printf()

  • fwrite()

  • fread()

  • perror()

  • strerror()

实验需求

  • 输入与输出相同： 如果输入文件和输出文件是同一个文件

    • 打印错误消息 “Input and output file must differ” 并以返回码 1 退出。

  • 字符串长度： 你不能对一行的长度做任何假设。因此，你可能需要读取非常长的输入行…

  • 文件长度： 你不能对文件的长度做任何假设，即文件可能非常长。

  • 无效文件： 如果用户指定了输入文件或输出文件，但当你尝试打开该文件（例如 input.txt）时因某种原因失败，应打印以下精确的错误消息：error: cannot open file 'input.txt'，然后以返回码 1 退出（即调用 exit(1);）。

  • 传递给程序的参数过多： 如果用户运行 reverse 时参数过多，打印 usage: reverse <input> <output> 并以返回码 1 退出。

  • 如何打印错误消息： 发生任何错误时，应使用 fprintf() 将错误打印到屏幕上，并将错误消息发送到 stderr（标准错误）而不是 stdout（标准输出）。在 C 代码中通过以下方式实现：fprintf(stderr, "whatever the error message is\n");

要退出，调用 exit(1)。你传递给 exit() 的数字（本例中为 1）可供用户查看程序是返回了错误（即返回非零值）还是正常退出（即返回了 0）。

对于读取输入文件，以下例程会让你的工作变得轻松：fopen()、getline() 和 fclose()。

对于打印（到屏幕或到文件），使用 fprintf()。注意，通过将 stdout 传递给 fprintf() 可以很容易地写入标准输出；通过传入 fopen 返回的 FILE * 也可以很容易地写入文件，例如：fp=fopen(...); fprintf(fp, ...);。

wcat

wcat 程序的功能是读取用户指定的文件并打印文件内容

依赖例程

  • fopen()

  • fgets()

  • fclose()

  • printf()

  • perror()

  • strerror()

实验需求

  • 你的程序 wcat 可以在命令行上使用一个或多个文件调用；
    • 它应该依次打印出每个文件。
  • 在所有非错误情况下，wcat 应以状态码 0 退出
    • 通常通过从 main() 返回 0（或调用 exit(0)）。
  • 如果命令行上没有指定任何文件，wcat 应直接退出并返回 0。
    • 注意，这与普通 UNIX cat 的行为略有不同（如果你愿意，可以找出不同之处）。
  • 如果程序尝试 fopen() 一个文件并失败，应打印精确的消息 “wcat: cannot open file”（后跟换行符）并以状态码 1 退出。如果命令行上指定了多个文件，应依次打印文件直到文件列表结束或遇到打开文件错误（此时打印错误消息并退出 wcat）。

流程分析

  1. 打开文件

    使用 fopen()

    • 未指定文件
      • 直接退出，返回0
    • 指定文件
      • 指定多个文件
        • 依次打印文件列表，直到 打开失败
      • 打开失败
        • 打印精确消息 “wcat: cannot open file\n”
        • 以状态码 1 退出
      • 打开成功：向下进行
  2. 读取文件内容

    使用 fgets()

    • 可以打开多个文件，按照 **argv 的传参顺序进行提取
      • 需要 (buffer_ptr, buffer_size, FILE*)
  3. 输出文件内容

    使用 printf()

  4. 关闭文件

    使用 fclose()

C++ Boost::Log

发表于 2026-04-03 更新于 2026-04-10

Log

一个高度模块化和可扩展的C++日志库，旨在为应用程序提供灵活的日志记录功能。

Boost::Log Refference

设计理念

提供 高度模块化 和 可扩展 的日志处理。各个模块可以独立使用和替换，并且能够添加自定义功能。支持宽/窄字符记录。

三层架构

  1. 日志数据收集层
    • 负责从程序中处理收集的日志事件
  2. 收集数据处理层
    • 负责处理收集到的日志事件
  3. 连接前两层的中央枢纽
    • 协调管理日志在上两层之间的流转

设计要点： 这个架构的核心优势是正交性，即日志的收集、处理方式和日志内容本身是相互独立的。这种设计让库不仅能用于传统日志记录，还能用于向用户发出通知、累积统计信息等场景。

关键组件

构成以上三层的具体组件：

  • 日志源 (Logging Sources)： 日志事件的发起者 。最常见的是日志记录器 (Logger)，它提供了类似 std::ostream 的流式接口来 格式化 日志消息。库也支持其他类型的源，如捕获子进程的控制台输出等。

  • 属性与属性值 (Attributes and Attribute Values)： 日志记录的元数据 。一个属性本质上是获取某个值的函数（例如获取当前时间），其返回的值就是 属性值 ，会在日志处理的后续阶段使用。属性分为 三种 类型：

    • 全局属性 ：附加到 所有 日志记录，优先级最低。

    • 线程特定属性 ：仅附加到 特定线程 生成的记录。

    • 源特定属性 ：由日志源维护 ，仅附加到通过 该源生成的记录 ，优先级最高。

    • 当同名属性出现在多个集合中时，优先级高的属性会覆盖优先级低的。

  • 日志核心 (Logging Core)： 中央枢纽的具体实现。它负责接收日志事件，并应用全局过滤器来决定哪些日志记录可以继续传递。核心被实现为 单例 (Singleton) ，可通过 boost::log::core::get() 访问。

  • 过滤器 (Filters)： 核心的一部分功能。过滤器用于在核心层面 筛选 日志记录，决定其是否被处理。

  • 接收器 (Sinks)： 日志处理的最终目的地。它是处理层的核心，负责将日志记录 格式化并输出到文件 、控制台、网络等不同目标。接收器采用前后端分离的设计：

    • 前端 (Frontend)： 负责接收日志记录，并决定同步、异步等传递策略。

    • 后端 (Backend)： 负责执行实际的格式化操作并将日志写入最终目标。
