wcat

wcat 程序的功能是读取用户指定的文件并打印文件内容

依赖例程

  • fopen()

  • fgets()

  • fclose()

  • printf()

  • perror()

  • strerror()

实验需求

  • 你的程序 wcat 可以在命令行上使用一个或多个文件调用；
    • 它应该依次打印出每个文件。
  • 在所有非错误情况下，wcat 应以状态码 0 退出
    • 通常通过从 main() 返回 0（或调用 exit(0)）。
  • 如果命令行上没有指定任何文件，wcat 应直接退出并返回 0。
    • 注意，这与普通 UNIX cat 的行为略有不同（如果你愿意，可以找出不同之处）。
  • 如果程序尝试 fopen() 一个文件并失败，应打印精确的消息 “wcat: cannot open file”（后跟换行符）并以状态码 1 退出。如果命令行上指定了多个文件，应依次打印文件直到文件列表结束或遇到打开文件错误（此时打印错误消息并退出 wcat）。

流程分析

  1. 打开文件

    使用 fopen()

    • 未指定文件
      • 直接退出，返回0
    • 指定文件
      • 指定多个文件
        • 依次打印文件列表，直到 打开失败
      • 打开失败
        • 打印精确消息 “wcat: cannot open file\n”
        • 以状态码 1 退出
      • 打开成功：向下进行
  2. 读取文件内容

    使用 fgets()

    • 可以打开多个文件，按照 **argv 的传参顺序进行提取
      • 需要 (buffer_ptr, buffer_size, FILE*)
  3. 输出文件内容

    使用 printf()

  4. 关闭文件

    使用 fclose()

wgrep

wgrep 程序的功能是在行中查找用户指定的搜索词

依赖例程

  • fopen()

  • fgets()

  • fclose()

  • getline()

  • printf()

  • perror()

  • strerror()

实验需求

  • 你的程序 wgrep 总是接收一个搜索词和零个或多个要 grep 的文件（因此可能不止一个文件）。它应逐行检查搜索词是否存在于该行中；如果是，则应打印该行，如果不是，则应跳过该行。

  • 匹配区分大小写。因此，如果搜索 foo，包含 Foo 的行将不匹配。

  • 行可以任意长（也就是说，你可能会在遇到换行符 \n 之前看到很多很多字符）。wgrep 即使在非常长的行上也应正常工作。为此，你可能想查阅 getline() 库调用（而不是 fgets()），或者自己实现。

  • 如果 wgrep 没有收到任何命令行参数，应打印 “wgrep: searchterm [file …]”（后跟换行符）并以状态 1 退出。

  • 如果 wgrep 遇到无法打开的文件，应打印 “wgrep: cannot open file”（后跟换行符）并以状态 1 退出。

  • 在所有其他情况下，wgrep 应以返回码 0 退出。

  • 如果指定了搜索词但没有指定文件，wgrep 应正常工作，但不是从文件读取，wgrep 应从标准输入读取。这样做很容易，因为文件流 stdin 已经打开；你可以使用 fgets()（或类似例程）从中读取。

  • 为简单起见，如果传入的搜索字符串是空字符串，wgrep 可以选择不匹配任何行或匹配所有行，两者都可以接受。

流程分析

  1. 打开文件

    使用 fopen()

    • 未指定参数
      • 输出回显
      • 状态码 1 退出
    • 未指定文件
      • 从标准输入中读取
      • getline(&line, &size, stdin)
    • 指定文件
      • 指定多个文件
        • 依次打印文件列表，直到 打开失败
      • 打开失败
        • 打印精确消息 “wgrep: cannot open file\n”
        • 以状态码 1 退出
      • 打开成功：向下进行
  2. 读取文件内容

    使用 fgets()

    • 可以打开多个文件，按照 **argv 的传参顺序进行提取
    • 严格按行读取
  3. 输出匹配文件行

    使用 printf()

    • 匹配区分大小写
  4. 关闭文件

    使用 fclose()

wzip 与 wunzip

wzip-wunzip 程序的功能分别是进行文件的压缩(wzip)与解压缩(wunzip)
使用 游程编码(REL) 进行压缩

依赖例程

  • fopen()

  • fgets()

  • fclose()

  • getline()

  • printf()

  • fwrite()

  • fread()

  • perror()

  • strerror()

实验需求

  • 正确调用应通过命令行向程序传递一个或多个文件

    • 如果没有指定文件，程序应以返回码 1 退出
    • 对于 wzip
      • 打印 “wzip: file1 [file2 …]”（后跟换行符）
    • 对于 wunzip
      • 打印 “wunzip: file1 [file2 …]”（后跟换行符）。

  • 压缩文件的格式必须与上述描述完全一致（每个游程为一个 4 字节整数后跟一个字符）。

  • 请注意，如果多个文件传递给 wzip，它们将被压缩为 一个 压缩输出，解压时也将变成 单个 未压缩的文本流（因此，最初有多个文件输入到 wzip 的信息将会丢失）。wunzip 也是如此。

流程分析

wzip

  1. 探查文件

    使用 fopen()

    • 无指定文件
      • 打印 “wzip: file1 [file2 …]”（后跟换行符）
      • 返回码 1 退出

  2. 压缩文件

    游程码为 4字节 整数 + 一个字符

    • 多文件同时压缩为一个文件
      • 注意，需要将所有文件预先读入同一个缓冲区，这样可以让前后文件相连，总体压缩

  3. 将压缩文件打印至标准输出

    使用 fwrite()

    • fwrite(&src, sizeof(src), n, FILE*)
      • 注意使用四字节存储游程，直接使用原数据类型指针，作为源指针&src

wunzip

  1. 探查文件

    使用 fopen()

  2. 读取压缩文件

    使用 fread()

    • 无指定文件
      • 打印 “wunzip: file1 [file2 …]”（后跟换行符）。
      • 返回码 1 退出

  3. 文件解压缩

    游程码为 4字节 整数 + 一个字符

    • 压缩为一个文件的多文件不进行分割

  4. 将解压的文件打印至标准输出

    printf()

    • 在使用 fread() 函数读后，需要使用 feof() 函数，探查是否已经读到文件结尾，此时可以退出读循环
