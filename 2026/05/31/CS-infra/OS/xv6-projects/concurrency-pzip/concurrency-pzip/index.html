pzip程序设计

实验目标

  • 熟悉 Linux pthreads。
  • 学习如何并行化程序。
  • 学习如何为高性能编程。

程序要求

  • 参考 initial-utilities/wzip (wunzip) 游程压缩 原理，编写 pzip 程序
  • 考虑使用 mmap 调用，将文件映射至内存中进行处理

合适例程

  • get_nprocs()

  • get_nproces_conf()

  • read()

  • fread()

  • mmap()

调用方式

程序流程

  1. 并行化配置

    • 使用 get_nprocs() 例程，获得逻辑线程总数，表征可用线程

  2. 并行化处理

    1. 文件读取

      • 使用 struct stat 获得文件列表中各个文件的大小
      • 调用 mmap 分配文件总大小空间，将文件内容写入该缓存
      • 将所有文件读入同一个缓冲区

    2. 文件压缩

      • 将 mmap 的总体缓冲进行均分分片处理
      • 针对各片建立mmap并行进行压缩
      • 子线程
        • task 处理压缩内存映射
      • 主线程
        • 将各片的缓冲进行合并，将各片的头、尾进行对比，合并相同字符信息
          • 合并各线程缓存 merge_zip_buf()

  3. 文件输出

    • 默认输出至 stdout

  4. 尾后处理

    • 释放线程资源
    • munmap 释放虚拟页映射缓冲

高性能编程理解

  • 通过 mmap 建立内存-文件映射，减少IO操作，转换为内存操作
  • 通过编写 thread_pool 统一管理任务线程
