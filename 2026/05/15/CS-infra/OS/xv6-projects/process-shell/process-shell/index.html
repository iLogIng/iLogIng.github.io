process-shell

发表于 2026-05-15

wish 程序设计

设计目标

  • 进一步熟悉 Linux 编程环境
  • 学习进程的 创建、销毁、管理
  • 接触 shell 中的必要功能

设计需求

两种模式

为每一个新命令创建一个进程

交互模式

提示符回显，循环，直接输入命令

允许用户直接输入命令。

批处理模式

无提示符回显，单次调用
从文件读取需要执行的命令

功能结构

参考例程

  • fopen()

  • fclose()

  • fprintf()

  • getline()

  • strsep()

  • strdup()

  • fork()

  • execv()

  • wait()

  • waitpid()

  • chdir()

  • getcwd()

  • access()

基本Shell

  • 运行在一个 while 循环中，反复请求输入，接收执行命令
  • 使用 exit 退出
  • 对于两种模式
    • 交互模式：在一个循环中
    • 批处理模式：通过读文件，执行命令
  • 退出方式
    • 遇到 EOF 后，调用 exit(0) 退出

路径

用户必须指定一个 路径变量 来描述要搜索可执行文件的目录集合

使用 绝对路径 或 相对路径 指定二进制文件

内建命令

每当shell接收一个命令时，检查该命令是否属于 内建命令

本项目实现：

  • exit
    • 退出shell程序
  • cd
    • 总是接收1个参数。更改当前目录.
  • path
    • 接收0/多个参数。覆盖shell搜索路径

重定向

使用 > 命令提供重定向功能

并行命令

通过 & 实现命令并行

一定同时运行 & 两侧的命令

程序错误

  • 打印 唯一且仅有的错误信息
  • 调用 exit(1) 退出

注意

  • 针对 空白字符 健壮
  • 检查系统调用的返回码

程序流程

  1. 解析程序命令行

    • 交互式
    • 批处理

  2. 根据命令行调用方式，处理程序

    循环在遇到 EOF 后，使用 exit(0) 退出

    • 交互式

      进行循环

      • 逐行读取命令
      • 输入 exit 内建命令退出
    • 批处理

      直接读取文件

      • 逐行读取命令
      • 完成后直接退出
        • EOF 后，退出

  3. 命令调用解析

    编写一个查找表结构进行处理

    • 内建命令
      • exit
      • path
      • cd
    • 系统命令
      • path 更新pash_route
    • 命令调用
      • 为系统命令添加路径前缀
        • access()
        • snprintf()
        • sys_cmd_full_cat();
      • 通过系统调用运行命令
        • fork()
        • execv()
        • wait()/waitpid()
        • cmd_executor();
          • fork() {
            • execv();
            • waitpid();
          • }

  4. 特性支持

    重定向
    并行命令

    • 添加 parse_line() 针对 > & 进行分词
    • 在shell_execute()中添加功能
    • 使用 & 进行分段

  5. 错误处理
