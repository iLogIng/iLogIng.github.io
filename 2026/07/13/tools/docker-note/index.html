Docker Note

Docker

容器化： 是一种 轻量级 的操作系统虚拟化方法。将应用、依赖、配置等集成为一个专有环境
具有：

  • 可移植性
    • 在一次构建与配置后，可以在任何装有Docker引擎的环境中一致运行
  • 更轻量化
    • 通过直接共享主机的操作系统内核，达到了相对于完整的虚拟机模拟程序， 更快的启动速度 、 更低的资源占用 的效果
    • 一台物理机可同时运行上千个容器
  • 资源隔离
    • 每个容器都拥有其自己独立的文件系统、网络和进程空间，互不干扰，安全性更高

Docker 的三个核心概念

  • 镜像 (Image)
    • 一个只读的模板，用于创建容器
  • 容器 (Container)
    • 镜像的运行实例，是运行实际应用的环境
  • 仓库 (Repository)
    • 集中存放和分享镜像的存储区域 如 Docker Hub

构建自己的容器镜像

为构建自己项目的一份镜像，需要编写 Dockerfile 与 .dockerignore 文件 控制镜像容器的构建
详细内容，请参考 官方教程

如 github.com/iLogIng/http_server 中的 Dockerfile 文件

  • FROM
    • 指定基础镜像，用于后续的指令构建
  • RUN
    • RUN <shell>
    • RUN ["exec", "arg1", "arg2", ...]
    • docker build 时运行
    • 在构建过程中在 镜像中 执行命令
    • 注意：Dockerfile的指令每执行一次都会在docker上新建一层，会导致镜像膨胀，可使用 && 符号连接命令
  • CMD
    • CMD <shell>
    • CMD ["exec", "arg1", "arg2", ...]
    • ENTRYPOINT ["exec", "arg1", "arg2", ...] \ CMD ["arg1", "arg2", ...]
    • docker run 时运行
    • 指定容器创建时的主要命令（可被覆盖）
  • ENTRYPOINT
    • ENTRYPOINT ["exec", "arg1", "arg2", ...]
    • 设置为固定参数
    • 设置容器创建时的主要命令（不可被覆盖）
  • EXPOSE
    • EXPOSE <port1> [port2, ...]
    • 指定容器运行时监听的特定网络端口
  • ENV
    • 在容器内部设置环境变量
  • ADD
    • 将文件、目录或远程URL复制到镜像中
  • COPY
    • COPY <host> <image>
    • 将文件或目录复制到镜像中
  • USER
    • USER <username>[:usergrade]
    • 指定后续指令的用户上下文

运行容器镜像

构建

docker build

部署运行

docker run
