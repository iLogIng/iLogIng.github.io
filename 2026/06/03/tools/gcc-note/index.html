编译参数选项

  1. 控制编译阶段与输出

    • -o <file>: 指定 输出文件名
      • 不填写，默认生成 a.out 可执行文件
    • -C: 预处理时不删除注释信息，一般与 -E 协同使用
    • -E: 仅 预处理 ，结果输出到 标准输出 需要进行重定向
    • -S: 仅 预处理 编译 ，不汇编和链接，生成 .s 文件
    • -c: 仅 预处理 编译 汇编 不链接 生成 .o 目标文件(.obj文件)
    • -v: 显示编译的详细过程，查看头文件搜索路径、库路径等，排查环境问题非常有用

  2. 警告与诊断

    • -Wall: 打开大多数常见的警告
    • -Wextra: 在 -Wall 基础上开启额外警告
    • -Werror: 把 所有警告当作错误 ，强制代码干净
    • -pedantic: 严格遵循ISO标准，发出所有强制性的诊断
    • -Wshadow: 变量遮蔽时警告，能帮你避免不少逻辑错误

  3. 调试与优化

    • -g: 生成调试信息，用于调试器(如 gdb)
    • -O0: 默认参数，不优化，调试时使用
    • -O1 -o: 基本优化
    • -O2: 进一步优化
    • -O3: 最高级别优化
    • -Os: 优化代码大小
    • -s: 剥离符号表，减小可执行文件体积
    • -Og: 在保持调试体验的前提下进行优化，调试时比 -O0 性能更好
    • -m32 / -m64: 生成32位或64位代码
    • -M: 生成文件关联信息。包含目标文件的所有依赖
      • gcc -M main.c 输出main.c中的头文件依赖
    • -MM: 忽略由 #include<file> 引入的依赖，即系统导入的依赖路径
      • 如 #include <iostream> 并不会输出
      • #include "mylib" 会输出
    • -MD: 将 -M 输出重定向到 .d 文件
    • -MMD: 将 -MM 输出重定向到 .d 文件

  4. 标准与兼容性

    • C
      • -std=c99
      • -std=c11
      • -std=c17
      • … 语言标准
    • C++
      • -std=c++11
      • -std=c++14
      • -std=c++17
      • -std=c++20
      • … 语言标准
    • -ansi 仅支持ANSI标准的C++语法禁止GNU扩展

  5. 头文件与库

    • -I<path>: 添加 头文件 搜索路径，无间隔
      • -I./include/
        • 在添加后，可以在*.c *.cpp文件中直接通过 #include "mylib.h" 引用 ./include/mylib.h 头文件
    • -nostdinc: 编译器不再使用系统默认的头文件目录中查找头文件定义，一般与 -Idir 联系使用
    • -L<path>: 添加 库文件 搜索路径，无间隔
      • -L/usr/lib/ 在 /usr/lib/ 路径下搜索库文件
    • -l<name>: 链接名为 lib<name>.so 或 lib<name>.a 的库
      • 即 -lib
      • -lpthread 链接pthread库
      • -lc 链接glibc库
    • -D<macro>[=value]: 预定义宏，例如 -DDEBUG
      • 相当于 #define MACRO [value]
    • -U<macro>: 取消宏定义
      • 相当于 #undef MACRO
    • -include <file>: 在预处理时强制包含一个文件，相当于在源码首行加 #include

  6. 链接与运行时

    • -static: 强制静态链接 或 生成 静态库
    • -shared: 生成 共享库 (Linux 下的动态链接) .so
    • -fPIC: 生成位置无关代码(Position Independent Code)，构建共享库时必须使用，该代码能在内存中的任意位置加载
    • -pthread: 链接时加入pthread库，并定义必要的宏
    • -Wl,<option>: 向链接器传递选项，例如 -Wl,-rpath,<path> 设置运行时库搜索路径
