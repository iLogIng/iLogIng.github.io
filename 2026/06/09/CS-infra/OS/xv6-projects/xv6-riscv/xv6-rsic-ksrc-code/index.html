在 xv6 中，程序的组织被设计为，使用 kernel/defs.h 作为 全局声明汇总头文件 ，集成了几乎所有内核函数的原型声明，便于各个源文件进行调用。

asm() 用于在C代码中嵌入汇编代码，即 内联汇编 技术。GCC编译器中 __asm()__ 与其等价。
volatile 关键字可以避免编译器优化内联的汇编代码
需要注意的是，内联的汇编代码通常与平台绑定，所以几乎无可移植性。

主程序 main.c

内核启动文件，是整个内核的入口函数

其内部结构很简单，只包含一对简单的 if-else 条件分支，并进行内核程序的直接初始化以及调度程序 scheduler() 的启用

其设计也很有意思，由于 xv6 是一个多核系统，则势必需要进行不同cpu核心的任务调度分配，
在 if 条件分支中，为了区分不同核心的不同动态，使用 cpuid() 调用，在主核心进行内核程序的初始化，
在 main.c 这一文件中还包含一个 volatile 修饰的静态全局变量 started ，使用 volatile 的作用是为了 **防止编译器优化 else 分支中的 while(started == 0) 这一忙等待循环。防止cpu读取寄存器中的缓存值，导致死循环。

cpuid() 在 kernel/defs.c 中声明，在 kernel/proc.c 中定义，

scheduler() 在 kernel/defs.c 中声明，在 kernel/proc.c 中定义
是 xv6 的核心调度循环，在启动后持续运行，不断在各个进程（或空闲状态）之间切换。每个hart都会进入该调度器。实际上，main.c程序在关机前都不会结束，这个就是整个 xv6 系统的主函数。

还可以看到 __atomic_thread_fence(__ATOMIC_SEQ_CST) 在设置 started 前后，保证在其前的 初始化操作都完成后 再将 started 赋值为1 (hart0 发布屏障)；在 忙等待 之后再进行 其它hart 的初始化。

保证了多核之间的内存可见性顺序 ，构成一个 无锁同步 模式

  • volatile 保障轮询的有效性(从内存中读取，以防止编译器优化，导致的寄存器缓存变量在忙等待中的死循环问题)
  • __atomic_thread_fence(__ATOMIC_SEQ_CST) 屏障保障数据的发布与订阅顺序（保证程序的逻辑顺序在编译时不受指令重排影响）。

volatile 关键字： 禁止编译器优化对变量的访问 ，表示不保证被修饰变量的值是否会在不知情的情况下被改变，在每次使用受 volatile 修饰的变量时都需要 从内存中 重新读取，而不使用寄存器中的缓存值。
注意： volatile 不能保证原子性 ，也 不保证顺序 ，仅能保证在使用时读取内存中的值

__atomic_thread_fence() 是 gcc 和 clang 提供的一个内置函数，用于生成一条硬件内存屏障指令，是 编译器内建的原子操作原语 ，
是编译器内置的 内存屏障 。强制编译器不会将屏障前后的访存指令跨屏障重排，同时会生成一条 CPU 硬件屏障指令，禁止硬件运行时对内存操作进行乱序执行，即保证在编译时指令重排操作不影响程序的逻辑顺序

hart 即 Hardware Thread(Tar T) 是 RSIC-V 指令集架构中的标准术语，表示 一个独立的硬件执行单元，或一个硬件线程
一个 CPU 核心 通常包含一套完整的运算单元
一个 hart 代表该核心上的一条独立指令执行流
大多数简单系统中，一个核心就是一个执行流，此时 一个核心 = 一个hart
若该线程支持超线程，一个物理核心可以同时运行多条指令流，此时 一个核心 = 多个hart

start.c

主要依赖 riscv.h 中riscv架构处理器寄存器操作函数、宏

  • start()
    • 启动程序，调起 main.c/main()
  • timerinit()
    • 初始化处理器时钟中断（定时器中断）

设置机器状态，启动分页，调起时钟中断初始化，返回到 main()

使用 w_mepc((uint64)main); 将 main() 函数的地址写入 mepc 寄存器，标识异常处理程序的起始地址

mepc ：是“机器异常程序计数器”（Machine Exception Program Counter）。
当 CPU 发生异常或中断而被迫跳转到 M-mode 的陷阱处理程序时，硬件会自动把被打断的指令地址（或下一条指令的地址）保存到 mepc 中。等内核处理完异常，执行 mret 指令时，CPU 就把 mepc 的值加载到普通 PC，从而实现精确返回。

mret : 机器模式返回寄存器 (Machine Return)，从机器模式的陷阱处理中返回

使用 asm volatile("mret"); 利用RISC-V的陷阱返回机制， mret 指令查看 mstatus 的 MPP (M Previous Privilege mode)字段，该字段已经通过以下方式设置

mret 检查 mepc 寄存器中的地址（即 main() 的地址）进行返回，跳转到 main.c/main() 函数
