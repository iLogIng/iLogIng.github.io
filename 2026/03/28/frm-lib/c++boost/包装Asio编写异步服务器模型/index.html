官方用例的问题

该服务器实现，将 TCP连接逻辑 与 业务处理逻辑 耦合在一个 *tcp_connection类 中。仅能作为一个简单的库使用用例。

  • 连接与业务强耦合
  • 难以复用代码进行功能扩充

改造方法

对于以上的问题，有以下 两种简洁的改进方法

是将连接与业务分离的两种方式

回调函数

使用回调lambda函数的方式解决业务写死在tcp_connection类中的问题

改造tcp_connection

  • 添加 handler_type = std::function<void(tcp_connection&)> 类型，承载函数回调
  • 保存一个函数对象 handler_type handler_
  • 在连接打开后直接调用 保存的函数对象

改造tcp_server

  • 同样的，保存一个 回调函数对象 handler_type handler_

使用方法

  • 需要一串客户的字节流请求，激活异步回调

适配的client代码

  • 发往服务器一段字节流
  • 回显服务器应答

继承接口类

定义一个名为 request_handler 的抽象类接口

request_handler抽象类接口

  • 其中handle纯虚函数接口的读写逻辑由子类实现

改造tcp_connection类

  • 使用 shared_ptr 持有指向 request_handler 对象的指针（唯一需要更改之处）

改造tcp_server类

  • 几乎无需更改

实际业务编写

  • 业务类继承抽象接口 request_handler
  • 业务类重写 handle 方法
